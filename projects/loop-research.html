Finding new ways to organize user research data

DESIGN, RESEARCH, USABILITY TESTING

Quick Summary

One of my biggest projects at Looppanel was to figure out a way to organize research data across calls which would make the life of a UX researcher easier.

Through research, we realized people constantly converge and diverge at every step to synthesize the data. To replicate this mental model, we prototyped and tested a few variations with the people so they could get to insights faster.

Context

In our conversations with researchers across the industry, we found that organizing and analysing research data is one of their biggest pain points. All of them mentioned the need to be able to get to insights faster. Since all of them had a different way of working and gathering this data, we wanted to create a way for them to use the tool as they deem fit.

To make getting to insights faster, we introduced a question script across the product for easier segmentation of data which could lead to faster analysis and insights.

We put a lot of thought into the question script interface, from the script creation, to viewing call data, all up to the point of analysis. The interface is flexible and easy to use so that people can use it in their research process.

Updates to the home screen

  1. Updated horizontal nav bar - As a information architechture rehaul exercise we made the nav bar at top to reflect the file structure.
  2. Recent Calls - Quick way to get into the last call and start the workflow.
  3. Link paste to start a call - Easy way to invite the looppanel note taker into the zoom call.
  4. Folders/Projects - All the calls would be put into projects so it's easier to attach them to a discussion guide/question script.

The home page was redesigned to make it look like a file explorer & have touchpoints to get into the workflow quickly.

Updates to the note taking screen

  1. Option to invite collaborators - We made the note taking screen a collaborative space where more than 1 person(s) can take notes simultaneously.
  2. Question script - Space to take notes directly within the question.
  3. Space for new note - Pressing enter creates a new space to type a new note.

The additions to the note taking screen were made in accordance with the research data from conversations with people and the direction company wanted to take w.r.t the question script.

Updates to the call screen

  1. CTA to Analysis page - Easier way to jump to the analysis page to see data across calls for the same question script
  2. Question script - Notes taken under specific questions in accordance with the question script used during note taking.
  3. Call notes with tags and other options - New view in which a summary can be added on top of the note taken during user interview. Other options show that the note can be converted to a video clip* or tagged.
  4. Overall uniform view - Cleaner view to add tags, see the time stamp etc.

Earlier, every note had to be attached to a video clip. But through our research we realized people don't always want a video clip. Also, they wanted to see a concise version of the notes in their own words.

Updates to the Analysis Screen

  1. Questions from the script - Concise version of the question script within a project with all call data.
  2. Individual call notes in a questions
  3. Project Summary
  4. Individual call data

In this screen we tried to create a view in which data across calls can be viewed easily and tagged.
